<h2 id="concept">Concept</h2>
<p>Imperative programming is like following a recipe step-by-step. You give the computer explicit instructions about what to do and in what order, focusing on <em>how</em> to achieve a result.</p>
<h2 id="real-life-analogy">Real-Life Analogy</h2>
<p>Imagine you're teaching someone to make a sandwich:
1. Get two slices of bread
2. Spread butter on one slice
3. Add cheese on the buttered slice
4. Place the second slice on top
5. Cut the sandwich diagonally</p>
<p>Each step modifies the state (the sandwich being built) until you reach the final result.</p>
<h2 id="dart-example">Dart Example</h2>
<pre class="codehilite"><code class="language-dart">void main() {
  // Imperative approach to calculate sum of squares
  List&lt;int&gt; numbers = [1, 2, 3, 4, 5];
  int sum = 0;

  // Step-by-step instructions
  for (int i = 0; i &lt; numbers.length; i++) {
    int square = numbers[i] * numbers[i];
    sum = sum + square;
  }

  print('Sum of squares: $sum'); // Output: 55
}
</code></pre>

<p>In this example, we explicitly tell the computer to:
- Initialize a sum variable
- Loop through each number
- Calculate the square
- Add it to the sum</p>

<p>Imperative programming gives you explicit control over the execution flow, making it ideal for algorithms and scripts where you need precise control over how operations are performed. The trade-off is that you must specify every detail of the process, which can make code more verbose compared to other paradigms.</p>
