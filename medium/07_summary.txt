<h3 id="object-oriented-approach">Object-Oriented Approach</h3>
<pre class="codehilite"><code class="language-dart">class Product {
  String name;
  double price;

  Product(this.name, this.price);

  bool isAffordable() =&gt; price &lt; 50;

  @override
  String toString() =&gt; '$name (\$$price)';
}

class ShoppingCart {
  List&lt;Product&gt; products = [];

  void addProduct(Product product) {
    products.add(product);
  }

  List&lt;Product&gt; getAffordableProducts() {
    return products.where((p) =&gt; p.isAffordable()).toList();
  }

  double calculateTotal(List&lt;Product&gt; items) {
    return items.fold(0.0, (sum, product) =&gt; sum + product.price);
  }
}

void ooExample() {
  ShoppingCart cart = ShoppingCart();
  cart.addProduct(Product('Book', 25.0));
  cart.addProduct(Product('Laptop', 850.0));
  cart.addProduct(Product('Mouse', 15.0));
  cart.addProduct(Product('Keyboard', 45.0));
  cart.addProduct(Product('Monitor', 200.0));

  List&lt;Product&gt; affordable = cart.getAffordableProducts();
  double total = cart.calculateTotal(affordable);

  print('Affordable products: $affordable');
  print('Total: \$${total}');
}
</code></pre>

<h3 id="functional-approach">Functional Approach</h3>

<h2 id="multi-paradigm-programming-in-dart">Multi-Paradigm Programming in Dart</h2>
<p>Dart is a multi-paradigm language, meaning it supports multiple programming styles. This flexibility allows you to choose the best approach for each situation.</p>
<pre class="codehilite"><code class="language-dart">// Combining paradigms in Dart

// OOP: Define classes
class Task {
  String title;
  bool isCompleted;
  int priority;

  Task(this.title, {this.isCompleted = false, this.priority = 1});

  void complete() {
    isCompleted = true;
  }
}

class TaskManager {
  List&lt;Task&gt; tasks = [];

  void addTask(Task task) {
    tasks.add(task);
  }

  // Functional: Use functional methods
  List&lt;Task&gt; getHighPriorityTasks() {
    return tasks.where((task) =&gt; task.priority &gt;= 3).toList();
  }

  List&lt;Task&gt; getPendingTasks() {
    return tasks.where((task) =&gt; !task.isCompleted).toList();
  }

  // Declarative: Express what we want
  void displayTaskSummary() {
    var pending = tasks.where((t) =&gt; !t.isCompleted).length;
    var completed = tasks.where((t) =&gt; t.isCompleted).length;

    print('Tasks Summary:');
    print('Pending: $pending');
    print('Completed: $completed');
  }
}

void main() {
  TaskManager manager = TaskManager();

  // Imperative: Step-by-step task creation
  manager.addTask(Task('Write article', priority: 3));
  manager.addTask(Task('Review code', priority: 2));
  manager.addTask(Task('Deploy app', priority: 4));
  manager.addTask(Task('Team meeting', priority: 1));

  // Complete some tasks imperatively
  manager.tasks[0].complete();
  manager.tasks[3].complete();

  // Use functional approach to filter
  print('High Priority Tasks:');
  manager.getHighPriorityTasks().forEach((task) {
    print('- ${task.title} (Priority: ${task.priority})');
  });

  print('\n');
  manager.displayTaskSummary();
}
</code></pre>

<h3 id="when-to-use-each-paradigm">When to Use Each Paradigm</h3>
<p><strong>Object-Oriented Programming:</strong>
- Building complex systems with many interrelated components
- When you need to model real-world entities
- When code reusability through inheritance is beneficial
- Examples: Game development, GUI applications, simulation systems</p>
<p><strong>Functional Programming:</strong>
- Data transformation and processing pipelines
- When immutability and predictability are important
- Concurrent or parallel processing
- Examples: Data analysis, stream processing, financial calculations</p>
<p><strong>Imperative/Procedural Programming:</strong>
- Simple scripts and utilities
- Performance-critical sections where explicit control is needed
- When the algorithm is naturally sequential
- Examples: System scripts, batch processing, algorithm implementation</p>
<p><strong>Declarative Programming:</strong>
- User interface development
- Database queries
- Configuration and setup
- Examples: Flutter UI, HTML/CSS, SQL queries</p>

<h2 id="best-practices-for-multi-paradigm-development">Best Practices for Multi-Paradigm Development</h2>
<ol>
<li><strong>Choose based on the problem</strong>: Don't force a paradigm; let the problem guide you</li>
<li><strong>Be consistent within a module</strong>: Mixing paradigms in the same function can be confusing</li>
<li><strong>Favor immutability when possible</strong>: It makes code easier to reason about</li>
<li><strong>Use OOP for structure, functional for logic</strong>: Combine their strengths</li>
<li><strong>Keep functions pure when you can</strong>: Pure functions are easier to test and debug</li>
<li><strong>Document paradigm choices</strong>: Help others understand your approach</li>
</ol>

<h2 id="conclusion">Conclusion</h2>
<p>Programming paradigms are not competing philosophies but complementary tools in your development toolkit. Modern languages like Dart embrace multiple paradigms, allowing you to select the most appropriate approach for each challenge.</p>
<p>Just as a craftsperson needs different tools for different jobs, a skilled programmer understands when to use object-oriented design for structure, functional programming for data transformation, imperative programming for precise control, and declarative programming for clarity.</p>
<p>The key to mastery is not choosing one paradigm over others, but understanding the strengths of each and knowing when to apply them. As you continue your programming journey, experiment with different paradigms, recognize patterns in successful code, and develop an intuition for which approach will serve your specific needs best.</p>
<p>Remember: the best code is not the one that rigidly follows a single paradigm, but the one that is clear, maintainable, and solves the problem effectively using the right combination of approaches.</p>
